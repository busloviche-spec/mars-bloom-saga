## 1. Починка «Ошибка генерации» в сундуках (ИИ)

Проблема: при клике «Открыть сундук» серверная функция `generateChestSeeds` падает, а тост показывает лишь общий текст «Ошибка генерации», без деталей. Наиболее вероятные причины: OpenAI отклоняет `strict: true` json_schema из-за жёсткой валидации (все `number` без `minimum/maximum` OpenAI принимает, но реальный сбой чаще возникает при `temperature: 1.0` + strict, или сеть/квота), либо ошибка приходит, но не логируется — и мы не видим её ни в логах, ни в UI.

Что сделаю:

- `src/lib/ai-chest.functions.ts`
  - Логировать причину сбоя на сервере: `console.error("[chest] openai error", status, bodySnippet)` перед `throw`, чтобы было видно в server-function-logs.
  - Обернуть парсинг JSON в try/catch и кидать понятную ошибку (`"OpenAI вернул некорректный JSON"`).
  - Понизить `temperature` до `0.9` и убрать `strict: true` (оставить `response_format: { type: "json_object" }` + чёткая инструкция «верни JSON вида {seeds: [...]}») — это самый частый источник 400-х ошибок на `gpt-4o-mini` со сложной схемой. Валидация полей и клэмп значений уже есть в `.handler()`.
  - Проверять `!key` заранее и возвращать явное сообщение «OPENAI_API_KEY не задан».
- `src/components/game/ChestDialog.tsx`
  - В catch выводить `e.message` в тосте (`toast.error("Ошибка генерации", { description: msg })` — уже есть, но убедиться, что server-fn прокидывает `message`, а не `[object Response]`; при необходимости оборачивать вызов и брать `err?.message ?? String(err)`).
- Мелкое: заменить deprecated `.inputValidator()` на `.validator()` (см. предупреждение Vite), чтобы не спутать это с реальной ошибкой в будущем.

Если после этого в логах увидим конкретный код (401/429/400), правку доведём точечно; сейчас цель — сделать сбой видимым и убрать частую причину (strict schema).

## 2. Более «червяковый» вид вредителя

Сейчас червь — просто эмодзи 🪱 в круглой кнопке. Сделаю его визуально узнаваемым сегментированным червём, который извивается.

- `src/components/game/PlantCell.tsx`
  - Заменить одиночный `🪱` внутри `.worm-pest` на композицию из нескольких сегментов: голова (тёмно-красный кружок с глазом-точкой и маленьким «ртом») + 4–5 сегментов тела уменьшающегося размера, слегка перекрывающих друг друга, в розово-красной гамме.
  - Контейнер меняем с круглой кнопки на вытянутую капсулу (`rounded-full`, `w-16 h-6`), сегменты — `absolute` внутри, каждый со своей задержкой анимации `worm-wiggle` для эффекта волны.
  - Полоска `biteProgress` остаётся под червём.
- `src/styles.css`
  - Обновить/добавить keyframes: `worm-wiggle` (уже есть) — использовать разные `animation-delay` на сегментах; добавить `worm-crawl` (лёгкое покачивание всей капсулы ±2px по X, 1.2s infinite).
  - Классы `.worm-body`, `.worm-head`, `.worm-seg` для сегментов и головы, чтобы разметка в React оставалась компактной.
- Клик, звук `sfx.squash()`, бонусы, тост, `squashPest()` — без изменений.
